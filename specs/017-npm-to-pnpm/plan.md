# Implementation Plan: NPM to pnpm Migration

**Branch**: `017-npm-to-pnpm` | **Date**: 2025-12-29 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/017-npm-to-pnpm/spec.md`

## Summary

Migrate the monorepo's package manager from npm to pnpm. This involves updating configuration files (package.json, adding pnpm-workspace.yaml), generating a new lockfile (pnpm-lock.yaml), removing the old lockfile (package-lock.json), and updating all documentation and shell scripts to reference pnpm commands instead of npm.

**Language/Version**: TypeScript 5.7.2, Node.js >=18.0.0
**Primary Dependencies**: Turborepo 2.3.3, NestJS 10.4.15 (backend), React 18.3.1 + Vite 6.0.5 (frontend)
**Storage**: SQLite via better-sqlite3 (no changes needed for migration)
**Testing**: Manual testing (POC phase - no automated test suite)
**Target Platform**: Linux/macOS development environments
**Project Type**: Monorepo with 3 packages (backend, frontend, shared)
**Performance Goals**: N/A (tooling change, not runtime performance)
**Constraints**: Must maintain identical functionality to npm-based workflow
**Scale/Scope**: 17 quickstart.md files, 6 shell scripts, 1 README.md, 1 CLAUDE.md, 4 package.json files
